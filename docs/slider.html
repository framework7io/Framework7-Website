Slider

Framework7 comes with powerful slider component with hardware accelerated transitions and great native behavior. It is built in a way similar to our well known Swiper.

Slider HTML Layout

Slider HTML layout is pretty simple:

<!-- Slider container -->
<div class="slider-container">
    <!-- Slides wrapper -->
    <div class="slider-wrapper">
        <!-- Slides -->
        <div class="slider-slide">Slide 1</div>
        <div class="slider-slide">Slide 2</div>
        <div class="slider-slide">Slide 3</div>
        ... other slides ...
    </div>
    <!-- Pagination, if required -->
    <div class="slider-pagination"></div>
</div>

Where:

  • slider-container - main slider container with slides and paginations. Required element

    • slider-wrapper - additional wrapper for slides. Required element

      • slider-slide - single slide element. Could contain any HTML inside

    • slider-pagination - container with pagination bullets. Note, that pagination bullets will be created automatically. Optional element

Initialize Slider with JavaScript

Now, when we have Slider' HTML, we need to initialize it. We need to use related App's method:

myApp.slider(sliderContainer, parameters) - initialize slider with options

  • sliderContainer - HTMLElement or string (with CSS Selector) of slider container HTML element. Required.
  • parameters - object - object with Slider parameters. Optional.
  • Method returns initialized Slider instance

For example:

var mySlider = app.slider('.slider-container', {
    speed: 400,
    spaceBetween: 100
});
Note that Slider container should be in DOM on a moment of initialization. So if you use slider not on home page, you need to initialize it within pageInit event

Slider Parameters

Let's look on list of all available parameters:

Parameter Type Default Description
initialSlide number 0 Index number of initial slide.
spaceBetween number 0 Distance between slides in px.
slidesPerView number 1 Number of slides per view (slides visible at the same time on slider's container).
direction string 'horizontal' Could be 'horizontal' or 'vertical' (for vertical slider).
speed number 300 Duration of transition between slides (in ms)
preventClicks boolean true Set to "true" to prevent accidental unwanted clicks on links during swiping
preventClicksPropagation boolean true Set to "true" to prevent clicks event propagation on links during swiping
Pagination
pagination string / HTMLElement undefined String with CSS selector or HTML element of the container with pagination.
paginationHide boolean true Toggle (hide/true) pagination container visibility when click on Slider's container.
Autoplay
autoplay number - delay between transitions (in ms). If this parameter is not specified, auto play will be disabled
autoplayDisableOnInteraction boolean true Set to false and autoplay will not be disabled after user interactions (swipes), it will be restarted every time after interaction
Navigation Buttons
nextButton string / HTMLElement undefined String with CSS selector or HTML element of the element that will work like "next" button after click on it
prevButton string / HTMLElement undefined String with CSS selector or HTML element of the element that will work like "previous" button after click on it
indexButton string / HTMLElement undefined String with CSS selector or HTML element of the element that will work like "index" button after click on it. For example if you use pagination and pass here ".slider-pagination-bullet", then click on each bullet will produce transition to the slide with same index number as the index number of clicked bullet.
Callbacks
onSlideChangeStart(slider) function Callback function, will be executed in the beginning of animation to other slide (next or previous). Receives slider instance as an argument.
onSlideChangeEnd(slider) function Callback function, will be executed after animation to other slide (next or previous). Receives slider instance as an argument.
onTouchStart(slider, event) function Callback function, will be executed when user touch Slider. Receives slider instance and 'touchstart' event as an arguments.
onTouchMove(slider, event) function Callback function, will be executed when user touch and move finger over Slider. Receives slider instance and 'touchmove' event as an arguments.
onOppositeTouchMove(slider, event) function Callback function, will be executed when user touch and move finger over Slider in direction opposite to specified in "direction" parameter. Receives slider instance and 'touchmove' event as an arguments.
onSliderMove(slider, event) function Callback function, will be executed when user touch and move finger over Slider and move it. Receives slider instance and 'touchmove' event as an arguments.
onTouchEnd(slider, event) function Callback function, will be executed when user release Slider. Receives slider instance and 'touchend' event as an arguments.
onClick(slider, event) function Callback function, will be executed when user click/tap on Slider after 300ms delay. Receives slider instance and 'touchend' event as an arguments.
onTap(slider, event) function Callback function, will be executed when user click/tap on Slider. Receives slider instance and 'touchend' event as an arguments.
onDoubleTap(slider, event) function Callback function, will be executed when user double tap on Slider's container. Receives slider instance and 'touchend' event as an arguments.
Namespace
slideClass string 'slider-slide' CSS class name of slide
slideActiveClass string 'slider-slide-active' CSS class name of currently active slide
slideNextClass string 'slider-slide-next' CSS class name of slide which is right after currently active slide
slidePrevClass string 'slider-slide-prev' CSS class name of slide which is right before currently active slide
wrapperClass string 'slider-wrapper' CSS class name of slides' wrapper
bulletClass string 'slider-pagination-bullet' CSS class name of single pagination bullet
bulletActiveClass string 'slider-pagination-active' CSS class name of currently active pagination bullet

Slider Methods & Properties

After we initialize Slider we have its initialized instance in variable (like mySlider variable in example above) with helpful methods and properties:

Properties
mySlider.params Object with passed initialization parameters
mySlider.container Dom7 element with slider container HTML element
mySlider.slides Dom7 collection of slides HTML elements
mySlider.activeSlideIndex Index number of currently active slide
mySlider.previousSlideIndex Index number of previously active slide
mySlider.isFirst true if slider on most "left"/"top" position
mySlider.isLast true if slider on most "right"/"bottom" position
Methods
mySlider.slideNext(); Run transition to next slide
mySlider.slidePrev(); Run transition to previous slide
mySlider.slideTo(index, speed);
  • index - number - index number of slide
  • speed - number - transition duration (in ms)
run transition to the slide with index number equal to 'index' parameter for the duration equal to 'speed' parameter.
mySlider.updateSlides(); This method will recalculate number of slides and their offsets. Useful after you add/remove slides with JavaScript
mySlider.updateSize(); Recalculate slider and slides sizes. Useful after you change slider size with JavaScript
mySlider.detachEvents(); Detach all events listeners
mySlider.attachEvents(); Atach all events listeners again
mySlider.destroy(); Destroy slider instance and detach all events listeners

Initialize Slider with HTML

If you don't need Slider callback you can initialize slider using HTML without JavaScript. You can do that just by adding additional "slider-init" class to slider-container

<!-- slider-init to initialize slider automatically -->
<div class="slider-container slider-init">
    <div class="slider-wrapper">
        <div class="slider-slide">Slide 1</div>
        <div class="slider-slide">Slide 2</div>
        <div class="slider-slide">Slide 3</div>
        ... other slides ...
    </div>
</div>

But what about Slider parameters. In this case we need to pass them in data- attributes:

<!-- slider properties in data- attributes -->
<div class="slider-container slider-init" data-speed="400" data-spaceBetween="40" data-pagination=".slider-pagination">
    <div class="slider-wrapper">
        <div class="slider-slide">Slide 1</div>
        <div class="slider-slide">Slide 2</div>
        <div class="slider-slide">Slide 3</div>
        ... other slides ...
    </div>
    <div class="slider-pagination"></div>
</div>

Or we can pass all slider parameters in valid JSON format in data-slider attribute:

<!-- slider properties in JSON format in data-slider attribute -->
<div class="slider-container slider-init" data-slider="{'speed':'400', 'spaceBetween':'40', 'pagination': '.slider-pagination'}">
    <div class="slider-wrapper">
        <div class="slider-slide">Slide 1</div>
        <div class="slider-slide">Slide 2</div>
        <div class="slider-slide">Slide 3</div>
        ... other slides ...
    </div>
    <div class="slider-pagination"></div>
</div>

Access to Slider's Instance

If you initialize Slider using HTML it is still possible to access to Slider's instance. It is "f7Slider" property of slider's container HTML element:

var mySlider = $$('.slider-container')[0].f7Slider;
 
// Now you can use all slider methods like
mySlider.slideNext();
